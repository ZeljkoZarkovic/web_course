Šta je XML?

Extensible Markup Language

Šta je to XML?

XML (Extensible Markup Language) je standardni, otvoreni i nezavisni format za opis i razmenu podataka na Web-u. On ne samo da čuva informacije, već i njihovu strukturu i značenje - zato se često opisuje kao “informacija o informaciji”. XML koristi tagove (oznake) da jasno definiše sadržaj, što omogućava i ljudima i računarima da lako čitaju i obrađuju podatke. Dokument je tekstualan, samoopisujući i odvojen od načina prikaza (npr. putem StyleSheet-a). Prednost XML-a je proširivost - korisnici mogu kreirati sopstvene oznake i prilagoditi jezik različitim potrebama. Koristi se za razmenu podataka između različitih sistema i baza, posebno u distribuiranim okruženjima.

    Tehnologije za: Specifikaciju logičke strukture XML dokumenata
  • DTD / XSD - definišu strukturu XML dokumenata
  • XSLT - transformacija XML sadržaja
  • XPath / XQuery - pretraga i upiti nad XML podacima
  • SOAP / WSDL - komunikacija i opis Web servisa

XML je fleksibilan, razumljiv i pouzdan način za razmenu podataka između različitih aplikacija i platformi.

    XML parseri:
  • DOM - Document Object Model
  • SAX- Simple API for XML Prikaz teksta
  • css - Cascading Style Sheets
  • XIink - XML Linking Language je jezik koji opisuje standardni način kreiranja hiperlinkova u XML dokumentima
  • XFragments
  • Xpointer - XML Pointer Language je jezik koji opisuje standardni način na koji hiperlinkovi pokazuju na određeno mesto unutar XML dokumenta
  • XForms - je sistem za pravljenje obrazaca zasnovan na XML-u
    Standarni zasnovani na XML-u
  • SOAP - Simple Object Access Protocol - jednostavan protokol baziran na XML-u koji omogućuje aplikacijama razmenu podataka u tekstualnom obliku preko HTTP protokola
  • WSDL -Web Service Definition Language - je jezik baziran na XML-u koji omogućuje opis web servisa
  • UDDI - Universal Description, Discovery and Integration Protocol

Istorija XML-a
Generalized Markup Language (GML) IBM je 60-ih razvio GML da bi olakšao upravljanje velikom količinom dokumentacije. Označavanjem sadržaja posebnim tagovima omogućeno je višestruko korišćenje istih podataka u različite svrhe.
Standard Generalized Markup Language (SGML) Iz GML-a je 80-ih nastao SGML, međunarodni standard (ISO 8879:1986) za strukturisanje dokumenata. Iako vrlo moćan, bio je složen i skup, pa se koristio uglavnom u velikim kompanijama i državnim institucijama.
HyperText Markup Language (HTML) , izveden iz SGML-a, namenjen je prikazu sadržaja na Web-u pomoću fiksnog skupa oznaka. Njegov nedostatak je što ne razdvaja sadržaj od forme, što kasnije rešava XML.
Extensible Markup Language (XML) XML je razvijen 90-ih od strane W3C kao jednostavniji i proširiv jezik koji kombinuje prednosti HTML-a i SGML-a. Ne zamenjuje HTML već ga dopunjuje i koristi se za razmenu, čuvanje i opis podataka. Standardizovan je 1998. kao XML 1.0.
XML atributi - Kao i u HTML-u, XML elementi mogu imati atribute u otvarajućem tagu koji daju dodatne informacije o elementu. Vrednosti atributa uvek moraju biti unutar navodnika, jednostrukih ili dvostrukih: <airline name="WizzAir"> ili <airline name='WizzAir'> Podaci se mogu čuvati i kao element: <airline>WizzAir</airline> ili kao atribut: <user airline="WizzAir">
XML parser je program koji čita i obrađuje XML dokumente. Ugrađen je u većinu pregledača (npr. Internet Explorer, Edge). Njegov zadatak je da rasčlani XML fajl i napravi hijerarhijsku strukturu podataka (XML stablo).

    Parser:
  1. Čita zaglavlje dokumenta (npr. <?xml version="1.0" encoding="UTF-8"?>)
  2. Prepoznaje početni (glavni) tag
  3. Obradjuje elemente i njihove atribute
  4. Formira odnose između elemenata (roditelj-dete)
  5. Ponavlja proces dok se ceo dokument ne obradi

Parseri omogućavaju ispravno čitanje i razumevanje XML strukture od strane aplikacija i sistema.
XML rečnici - XML je brzo postao osnova za razvoj brojnih specijalizovanih jezika koji se koriste u različitim oblastima. Neki od poznatih XML rečnika su:

  • SVG - opis vektorske grafike
  • MathML - opis matematičkih izraza
  • RDF - model za predstavljanje i obradu podataka na Web-u
  • XHTML - HTML u XML formatu
  • SMIL - opis multimedijalnih dokumenata
  • WML - XML jezik za prikaz podataka na mobilnim uređajima (WAP protokol)

AJAX i XML Asynchronous JavaScript i XML omogućava razmenu podataka između pregledača i servera bez ponovnog učitavanja stranice. Zasniva se na JavaScript-u, XML-u, HTML-u. Pomoću XMLHttpRequest objekta, AJAX omogućava brže i interaktivnije web aplikacije. U suštini, JavaScript reaguje na događaje na stranici, menja sadržaj pomoću DOM-a, i asinhrono komunicira sa serverom. Time aplikacije postaju dinamičnije i efikasnije.
XML baze podataka - XML dokumenti se mogu čuvati kao obične datoteke ili u bazama podataka. XML baze omogućavaju jednostavno skladištenje, pretragu i razmenu podataka između različitih sistema.

    Zbog hijerarhijske strukture XML-a, razvijeni su posebni sistemi za njegovo efikasno skladištenje:
  • Flat files - XML se čuva direktno u datotekama; pogodno za male količine podataka.
  • Relacione baze (XML enabled) - klasične baze (npr. Oracle, SQL Server) sa podrškom za XML. Omogućavaju upite pomoću XPath i DOM API-ja.
  • Native XML baze - baze posebno kreirane za XML. Čuvaju dokumente u izvornom obliku, podržavaju XQuery i omogućavaju indeksiranje i brzu pretragu.

WEB servisi

Web servisi uključuju tri glavne uloge: PROVAJDER SERVISA, REGISTAR SERVISA i KORISNIK SERVISA. Tri glavne operacije vezane za web servise su: publikovanje, pronalaženje i povezivanje (Bininding).

SOAP zahtev - U sledećem primeru, šalje se SOAP zahtev za metodu web servisa pod nazivom calculateCarPayment i koji prosleđuje tri različita argumenta.

REST je šablon za dizajn web servisa (design pattern). Predstavlja alternativni način kreiranja web servisa.

    Konkretna implementacija REST web servisa sledi četiri glavna dizajn principa:
  • Eksplicitno korišćenje HTTP metoda
  • Nezavisnost od konekcije (stateless)
  • Izlaganje resursa puter URI-a
  • Koristiti XML i/ili JavaScript Object Notation (JSON), za prenos podataka.
    Bezbednost web servisa obuhvata niz mehanizama koji omogućavaju zaštitu podataka i kontrolu pristupa u komunikaciji između klijenta i servera. Ključni aspekti bezbednosti su:
  • Autentifikacija - identitet - Ko je korisnik?
    - Kako dokazujemo da je neko stvarno to što tvrdi?
  • Autorizacija - kontrola pristupa - Šta korisnik može da uradi?
  • Poverljivost - enkripcija - Kako da sprečimo uvid u podatke od neovlašćenih lica?
  • Integritet - onemogućavanje izmena - Kako da sprečimo izmenu poruka na putu od pošiljaoca do primaoca?
    Tehnologije i mehanizmi za bezbednost Web servisa:
  • Autentifikacija - identitet - HTTP autentifikacija (basic, digest)
    - SSL digitalni sertifikati
  • Autorizacija - kontrola pristupa - Single Sign-On (SSO)
    - SAML (Security Assertion Markup Language)
  • Poverljivost - enkripcija - SSL enkripcija
    - XML Encryption
  • Integritet - onemogućavanje izmena - XML Digital Signatures
