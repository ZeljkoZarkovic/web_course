Šta je HTML5 i CSS3?

Hyper Text Markup Language i Cascading Style Sheets

Šta je to XHTML?

XHTML je verzija HTML-a modifikovana da odgovori XML standardu. Dizajniran da odvoji sadržaj od prezentacije: Sadržaj u XHTML-u i Prezentacija pod kontrolom Cascading Style Sheets (CSS). On je proširiv, mogu da se definišu dodatni elementi. XHTML razlikuje velika i mala slova. Svi elementi moraju da imaju početni tag i krajnji tag. Prazni elementi sadrže sopstveni krajnji tag. Vrednosti atributa moraju da budu pod znacima navoda.

    Istorija HTML/XHTML
  • 1992 - HTML prvi put definisan
  • 1994 - HTML 2.0
  • 1995 - Netscape specifični ne standardni HTML
  • 1996 - HTML 3.2, kompromisna verzija
  • 1997 - HTML 4.0, odvaja sadržaj od prezentacije
  • 1998 - XML standard za pisanje Web jezika
  • 2000 - XHTML 1.0, HTML usklađen sa XML-om
  • 2002 - XHTML 2.0

Naredba DOCTYPE Deklariše određenu verziju HTML-a koja se koristi na stranici. Koristi je pretraživač za određivanje kako da se obradi stranica. Uvek je prva naredba u HTML datoteci. Svaki XHTML dokument mora da počne sa jednom !DOCTYPE dekleracijom.

    Tri glavna tipa DTD-a su:
  • Strict - Koristi se za definisanje strukture stranice, bez formatiranja (bez fontova, boja, veličine itd.). Takođe, koristi u kombinaciji sa CSS (Cascading Style Sheets) ako želite da definišete kako dokument treba da izgleda.
  • Transitional - Korišćenje sa standardnim HTML i/ili CSS. Omogućava zastarele HTML elemente.
  • Frameset - Koristi se ako dokument koristi HTML okvire.

Elementi - sastoje se iz tri dela: početni tag koji može da sadrži atribute, sadržaj i završni tag. Primer: <p id="intro">Dobrodošli</p> Za više informacija za <p> pogledati.
Atributi - Uvek se koriste samo u početnom tag-u elementa.

    Tri tipa:
  • Opcioni atributi: Varira sa tipom elementa
  • Standardni atributi: id, class, title, style, dir, lang, xml:lang
  • Događaj atributi: onclick, ondblclick, onmousedown, onmouseup, onmouseover, onmousemove, onmouseout, onkeypress, onkeydown, onkeyup. Koriste se u skriptama.

Prazni elementi - Neki elementi nemaju sadržaj pa samim tim nemaju ni završni tag: <img src="photo.jpg" /> ; <br /> ; <hr /> ; <link rel="stylesheet" type="text/css" href="main.css" /> ; Kod XHTML-a, koji zahteva završne tagove na svim elementima, jedan tag predstavlja i početni i završni tag.
<h1> , <h2>, <h3> itd - su naslovi (zaglavlja) na stranici. Predstavljaju glavnu temu, podnaslove i sl. na stranici. Važno je da se koriste na logičan način, koji u velikoj meri pomaže pomoćne tehnologije kao što su pretraživači govora koji predstavljaju na razumljiv način sadržaj stranice.
<p> tag - Paragraf se definiše <p> tagom. Web pretraživači automatski dodaju praznu liniju (red) iznad i ispod paragrafa. Navodi/citiranja (<blockquote>) koriste se za navođenje dužih (preko 4 reda) citata. Web pretraživači obično uvlače (na desno) taj deo teksta.

    Liste
  • Nesređene liste (liste nabrajanja - bullets)
    <ul> <li>One</li> <li>Two</li> </ul>
  • Uređene liste (numbered lists)
    <ol> <li>One</li> <li>Two</li> </ol>
    Označavanje teksta:
  • Podebljanja
    <b>text</b>
    <strong>text</strong>
  • Kurzivi
    <i>text</i>
    <em>text</em>


  • Drugi
    <sub>text</sub> subscript
    <sup>text</sup> superscript
    <del>text</del> deleted text

Grafika Grafika se postavlja korišćenjem img elementa. Alt atribut daje alternativni tekst koji opisuje grafiku u slučaju da grafika ne može biti prikazana ili korisnik ne može da vidi grafiku. <img src="picture.gif" alt="Suzzallo">

Anchors (hiperlinkovi) mogu da povežu stranicu sa bilo kojom datotekom na web-u. <a href="http://www.washington.edu/"> University of Washington </a>
Div ograđuje skup elemenata.
<div style="text-align: center;">
<h2> News </h2>
<p><a href="budget.html">Budget</a></p>
<p><a href="invest.html">Investment</a></p> </div>
Span ograđuje objekte (tekst, grafike) u okviru elementa.
<p>Ovaj tekst je <span style="color: red;">crvene boje</span> unutar paragrafa.</p>

Šta je to HTML5?

HTML5 je jezik za označavanje koji se koristi za strukturiranje i prikazivanje hipertekstualnih dokumenata na internetu. Predstavlja petu i poslednju glavnu verziju HTML-a, koja je povučena kao zvanična preporuka Svetske organizacije za standarde (W3C). Trenutna verzija HTML-a je poznata kao HTML Living Standard.
Novine kod HTML5 - Novi semantički elementi i atributi. Ugrađeni API koji pomažu u izgradnji web aplikacija. Pridodata podrška za rad sa audio i video datotekama. HTML5 će smanjiti potrebu za dodacima (plug-ins) web pretraživača.

Novi elementi u HTML5: <article>, <aside>, <audio>, <canvas>, <command>, <datalist>, <details>, <embed>, <figcaption>, <figure>, <footer>, <header>, <hgroup>, <keygen>, <mark>, <meter>, <nav>, <output>, <progress>, <rp>, <rt>, <ruby>, <section>, <source>, <summary>, <time>, <video>.
Elementi koji su izbačeni iz HTML5: <acronym>, <applet>, <basefont>, <big>, <center>, <dir>, <font>, <frame>, <frameset>, <isindex>, <noframes>, <s>, <strike>, <tt>, <u>.
Header elemenat može da se koristi da obuhvati uvodni sadržaj ili navigaciona pomagala koja su specifična za bilo koji pojedinačni deo stranice, ili koja se odnose na celu stranicu ili oboje. Iako se header elemenat najčešće postavljna na vrh stranice ili sekcije, njegova definicija je nezavisna od njegove pozicije. Nema ograničenja za njegovo korišćenje samo jednom po stranici.
Section elemenat predstavlja generičku sekciju dokumenta ili aplikacije. Sekcija, u ovom kontekstu, je tematski grupisan sadržaj, obično uz naslov. Neki primeri prihvatljivog korišćenja za elemenat section uključuju:

  • pojedinačne sekcije i tab-ove.
  • segmente strane "O nama” - na primer, stranica "O nama" za kompanije može da sadrži delove o istoriji kompanije, misiju, kao i njen tim.
  • različite delove stranice dužih “Pravila upotrebe“ (terms of servisis, na primer copyright itd.).
  • različite sekcije sajta za online vesti; na primer, članci mogu biti grupisani u sekcije koje pokrivaju sport, zabavu, ekonomske vesti, itd.

Article elemenat predstavlja samostalnu kompoziciju u dokumentu, stranicu, aplikaciju ili sajt i koja se, u principu, može nezavisno distribuirati ili upotrebiti - npr. kod syndication tehnologija (RSS ili ATOM). Upotreba: poruke na forumu, časopisi i novinski članci, tekstovi u blogu, komentari korisnika.
Nav elemenat predstavlja linkove za navigaciju. Takođe možete postaviti nav elemenat oko pasusa teksta koji sadrži navigacione veze za stranicu ili sekciju stranice. Neki programeri veruju da je ovaj element pogodan za paginacije ili dubinske veze (breadcrumbs), ili kod formi za pretragu koje predstavljaju osnovno sredstvo za navigaciju sajta (kao što je slučaj na Google-u).
Aside elemenat Ovaj elemenat predstavlja deo stranice koji je "tangencijalno" u vezi sa sadržajem oko aside elementa, a koji bi se moglo smatrati da je odvojen od sadržaja. Neke od mogućih upotrebe aside elementa obuhvataju sidebar, sekundarne liste veza, ili blok reklama.
Footer elemenat u skladu sa specifikacijom, predstavlja podnožje sekcije sadržaja koji je u najbližoj vezi. “Sekcija" sadržaja može da bude ceo dokument, ili može da bude sekcija, članak, ili aside elemenat. Često footer ćе sadržati informacije o autorskim pravima, lista srodnih veza, informacije o autorima i slični sadržaj koji obično smatramo da dolazi na kraju bloka sadržaja.
Figure i figcaption elementi - Ovi elementi mogu da se koriste za obeležavanje ilustracija, dijagrama, fotografija, lista koda itd, koji referenciraju iz glavnog sadržaja dokumenta, ali koji mogu da se premeste iz primarnog sadržaja na neku drugu lokaciju na stranici, na posebnu stranicu ili kao dodatak (appendix), a sve bez uticaja na sam tok dokumenta.

Progress i meter elementi
Progress element se koristi da opiše trenutni status promena procesa koji idu ka završetku, bez obzira da li je stanje završetka definisano.
Meter elemenat, međutim, predstavlja elemenat čiji opseg se zna, što znači da ima definisanu minimalnu i maksimalnu vrednost. Meter elemenat ima šest povezanih atributa. Pored maksimalne (max) vrednosti, takođe omogućava korišćenje minimalne (min), visoke (high), niske (low), i optimalne (optimum).
Time elemenat je specijalno kreiran da se nosi sa problemom da ljudi čitaju datume i vreme različito od mašina.
Pattern atribut omogućava da se obezbedi regularni izraz s kojim mora da se upari korisnikov ulaz da bi se smatrao validnim. Za bilo koji ulaz, gde korisnik može da unese tekst u slobodnoj formi, može da se ograniči koja sintaksa je prihvatljiva pomoću pattern atributa.
HTML5 Form atributi Disabled, Readonly, Multiple, Form, Autocomplete, Datalist, Autofocus.
HTML5 Audio i video - Uz HTML5, nema potrebe da korisnik preuzima dodatne programe (plug-ins ili plejeri) da bi video sadržaj, a video ili audio plejer je lako dostupan preko skriptovanja. Video na web-u se zasniva na formatima kontejnera i kodecima. Kontejner je omotač (wrapper) koji čuva sve potrebne podatke koji obuhvataju video datoteku kojoj se pristupa, slično kao što ZIP datoteka obavija ili sadrži fajlove. Video kontejner formati koje su relevantni za HTML5 su MPEG-4, Ogg i WebM. Video kodek definiše algoritam za kodiranje i dekodiranje video sadržaja. Kodek može da kodira tok podataka za prenos, čuvanje, ili enkripciju, ili može da ga dekodira za reprodukciju ili uređivanje. Video kodeci koji se najviše odnose na HTML5 video su H.264, Theora i VP8.

Šta je to CSS?

CSS (Cascading Style Sheets) je jezik za stilizovanje koji se koristi za definisanje izgleda i formatiranja web stranica napisanim u HTML-u, XML-u ili drugim jezicima za označavanje. CSS je ključna tehnologija na internetu, zajedno sa HTML-om i JavaScript-om. CSS Level 3 (CSS3) koristi za određivanje vizuelnog izgleda web stranica za razliku od HTML-a (HyperText Markup Language), koji je jezik za označavanje koji definiše strukturu dokumenta za distribuciju na webu. Za razliku od CSS1 i 2, jedinstvena i sveobuhvatna CSS3 specifikacija ne postoji. Umesto toga, umesto da nastoji da kreira i objavi celu specifikaciju odjednom, CSS Working Group je podelila specifikaciju u niz modula.

CSS3 boje - CSS3 takođe pruža i brojne druge opcije: HSL, HSLA i RGBA. Najznačajnija promena sa ovim novim tipovima boja je sposobnost definisanja poluprovidnih boja. Gradijenti su postepeni prelazi između dve ili više navedenih boja. U stvaranju gradijenta, možete odrediti više vrednosti boja između, koje se zovu color stops. Svaki color stop je sačinjen od boja i položaja; web pretraživač bledi boju od svake color stop tačke do sledeće i time stvarajući gladak (kontinuirani) gradijent-prelaz.
Primer koda Linearnog gradienta: <background-image: -moz-linear-gradient(30deg, #000, #FFF 75%, #000)>
Primer koda Radialnog gradienta: <background-image: -moz-radial-gradient(#FFF, #000)>
Za uključivanje fontova korišćenjem @font-face morate da:

  1. Postavite font datoteku na server u različitim formatima radi podrške različitih pretraživača.
  2. Imenovati , opisati i povezati se na taj font u @font-face pravilu.
  3. Uključiti naziv fonta u vrednost svojstva font-family, baš kao što biste uradili za sistemske fontove.

HTML5 API

Canvas - HTML5 canvas element pruža virtuelno platno za kreiranje bitmap slika. Canvas API omogućava interakciju sa canvas elementom korišćenjem JavaSript-a. Canvas je moguće koristiti za jednostavne crteže puput geometrijskih oblika, ali i naprednijih stvari poput animacija.
Geolocation omogućava svojim posetiocima da dele svoje trenutne lokacije. U zavisnosti od toga kako oni posećuju vaš sajt, njihova lokacija može se odrediti na bilo koji od sledećih načina: IP adresa, bežična mrežna konekcija, Toranj mobilnog operatera, GPS hardver na uređaju.
Offline Web aplikacije rade tako što koriste tehnologiju keširanja (application cache). Keš aplikacije mogu da uskladište ceo sadržaj sajta: sve JavaScript-ove, HTML i CSS, kao i sve vaše slike i resurse.

    Postoje tri koraka za pravljenje Offline Web aplikacija:
  • Kreirati cache.manifest fajl.
  • Obezbediti da se manifest file servisira sa ispravnim tipom sadržaja.
  • Usmeriti sve HTML datoteke u keš manifest.

Web Storage API definiše standard za način kako mogu da se podaci snime lokalno na računaru ili uređaju korisnika. Pomoću Web Storage, sada imamo standardizovanu definiciju o tome kako da se uskladišti do 5MB podataka kreiranih od strane sajtova ili web aplikacija. Postoje dve vrste HTML5 Web Storage: skladištenje sesije i lokalno skladište.
Novi Web Workers API omogućava da se pokrenu veliki skriptovi u pozadini bez prekida funkcionisanja web stranice ili web aplikacije. Web Socket definiše "protokol za dvosmernu komunikaciju sa udaljenim host-om.“
Drag and Drop API omogućava da se određeni elementi specificiraju kao draggable (mogu da se pomeraju), a zatim odredi šta treba da se desi kada se ti draggable elementi premeštaju na druge elemente na stranici.
